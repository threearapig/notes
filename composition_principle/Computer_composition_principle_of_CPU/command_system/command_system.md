# 指令系统



指令系统在计算机中的地位



> 用高级语言或汇编语言编写的程序，如果要在计算机上执行，必须要利用编译程序或者汇编程序
>
> 将高级语言或汇编语言的指令变成机器指令，才能够由计算机的硬件按序进行执行



机器指令：指计算机系统的 CPU 能够直接识别并且执行的操作命令

一个处理器能够执行的所有机器指令构成的集合，称为**指令集**



---



指令系统就是计算机硬件和软件的交界面

> 软件通过指令系统告诉计算机硬件做什么操作
>
> 计算机硬件通过指令系统将运算结果和硬件的状态返回给软件





## 机器指令

**指令的格式是什么？**

* 操作码：指明此指令做的是什么操作
* 地址码：对谁进行操作

> 为了编程的方便，在指令集当中，通常需要使用多种类型的寻址方式



**指令的字长**

> 一条指令的长度是多少位

* 固定字长
* 可变字长



### 指令的一般格式

> 逻辑的表示方式
>
> 操作码和地址码不一定集中式的存在一起

| 操作码字段 | 地址码字段 |
| :--------: | :--------: |

* 操作码字段反映了机器做什么操作，并且不止于此

  > 有很多的机器的指令集当中，操作码还要指出对什么样的数据做操作，有时还指出操作数的寻址方式

  操作码的长度固定

  * 用于指令字长较长的情况：RISC

  * 固定操作码是为了：译码过程的方便，译码电路的简单

  操作码的长度可变

  * 操作码分散在指令字的不同字段中
  
  > 为了支持操作码的长度可变，需要采用扩展操作码技术来扩展操作码的长度
