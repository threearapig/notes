## 注意事项


### 关于函数


#### 语句和分号

语句表示 **要执行的操作**  
> 编译器需要知道一条语句何时结束，另一条语句何时开始  


C++ 和 C 都使用终止符来标记一条语句的结束，终止符使用分号 `;` 来表示  
> 终止符是语句的结束标记，是语句的组成部分  
> 而不是语句之间的标记  
> 在 C++ 和 C 中不能省略分号  



通常，main() 被启动代码调用，而启动代码是由编译器添加到程序中的，是程序和操作系统之间的桥梁

预处理器  
预处理器是在程序进行主编译之前对源文件进行处理  
预处理器处理名称以 `#` 开头的编译指令  
不需要执行任何特殊的操作来调用该处理器，它会在编译的时候自动运行
```cpp
#include <iostream>
```

> 该编译指令导致预处理器将 iostream 文件的内容添加到程序中  
> 在源代码被编译之前，替换或添加文本  

> iostream 文件的内容将取代程序中的代码行 `#include <iostream>`   
> 原始文件没有被修改，而是将源代码文件和 iostream 组合成一个复合文件，编译的下一阶段将使用该文件

using 编译指令  
为什么需要名称空间：  
可能使用两个已封装好的产品，而它们都包含一个名为 wanda() 函数。  
此时在使用 wanda() 函数时，编译器将不知道指的是哪个版本  
名称空间让厂商能够将其产品封装在一个叫做名称空间的单元中，这样就可以用名称空间来指出想使用哪个厂商的产品  
因此，Microflop Industries 可以将其定义放到一个名为 Microflop 的名称空间中，其 wanda() 函数的全称为 `Microflop::wanda();`  
Piscine 公司的 wanda() 版本可以表示为 `Piscine::wanda();`  
```cpp
Microflop::wanda();
Piscine::wanda();
```

因此，iostream 中定义的用于输出的 cout 变量实际上是 `std::cout`，而 endl 实际上是 `std::endl`  

但是这种加前缀的方法有点烦，此时就需要使用 using：
```cpp
using namespace std;
```

这样就不必每次都加上 std:: 前缀了
