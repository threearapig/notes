## 寄存器


vim 寄存器一览：  
| 寄存器 | 用途                           |
|--------|--------------------------------|
| a - z  | 有名寄存器                     |
| "      | 无名寄存器                     |
| 0      | 复制专用寄存器                 |
| _      | 黑洞寄存器                     |
| +      | 加号寄存器（系统剪切板）       |
| =      | 表达式寄存器                   |
| %      | 此寄存器记录当前文件名         |
| #      | 此寄存器记录轮换文件名         |
| .      | 此寄存器记录上次插入的文本     |
| \:     | 此寄存器记录上次执行的 Ex 命令 |
| \/     | 此寄存器记录上次查找的模式     |




**vim 提供了几十组寄存器用于保存文本，而并非只使用系统单一的剪切板**  

---

#### vim 术语对照表


| 操作   | 说明 | operator |
|--------|------|:----------:|
| put    | 粘贴 | p        |
| yank   | 复制 | y        |
| delete | 剪切 | d        |

---

#### 引用指定寄存器

> vim 提供了一组 26 个英文字母命名的有名寄存器  
>
> 这意味着可以同时保存多达 26 段文本  

有名寄存器：`"a - "z`  

**用小写字母引用有名寄存器，会覆盖该寄存器的原有内容**  
**用大写字母引用有名寄存器，会将新内容添加到该寄存器的原有内容之后**  

---


**普通模式下的格式：**

```text
"{register}operator[motion]
```

例如：
```text
"add
```

> 将光标所在行的文本 delete 到 a 寄存器


**Ex 命令的格式：**  

```text
:operator {register}
```

例如：

```text
:delete a
```

> 将光标所在行的文本 delete 到 a 寄存器



#### 一些特殊寄存器


**无名寄存器**  

> vim 缺省使用的寄存器  

用**双引号**`"`引用无名寄存器  

```text
""operator[motion]
```

---

**复制专用寄存器**  

> 在没有指定寄存器进行 yank 时，不仅会将这段文本 yank 进无名寄存器，还会 yank 进复制专用寄存器  
>
> 这也放置无名寄存器中的内容被轻易覆盖

用 `"0` 引用复制专用寄存器  

```text
"0operator[motion]
```

---


**黑洞寄存器**  

> 删除该文本且不保存任何副本  
>
> 黑洞的意思就是进去了就没有了  


用**下划线**`_`引用黑洞寄存器

```text
"_operator[motion]
```

---


**系统剪切板**  

> 如果想从 vim 复制文本到外部程序（反之亦然），则必须使用系统剪切板  


**加号寄存器**与系统剪切板等效  
用加号 `+` 引用加号寄存器  

```text
"+operator[motion]
```

> 如果在外部程序中用剪切或复制命令获取了文本  
> 可以通过命令 `"+p` （或在插入模式下用 `<C-r>+`）  
> 将其粘贴到 vim 内部  

> 如果在 vim 的复制或删除命令之前加入 `"+`   
> 相应的文本将被捕获至系统剪切板  
> 将其粘贴到外部程序中  

---


**表达式寄存器**  

> 表达式寄存器中一开始不会存储文本，当从表达式寄存器获取内容时  
> vim 将跳到命令行模式，并显示提示符 `=`，之后便可以输入一段 vim 脚本表达式并按 `<CR>` 执行  
> 如果这段 vim 脚本返回的是字符串（或可被强制转换成字符串的数据），vim 将会使用它  

```text
"=operator[motion]
```

---


**其他寄存器**  

> vim 提供了几组可被隐式赋值的寄存器，这些寄存器被称作只读寄存器  


| 寄存器 | 内容               |
|--------|--------------------|
| "%     | 当前文件           |
| "#     | 轮换文件名         |
| ".     | 上次插入的文本     |
| ":     | 上次执行的 Ex 命令 |
| "/     | 上次查找的模式     |


> 从技术上讲，`/` 寄存器并非只读，可以用 `:let` 命令对其进行显示的赋值  




#### 高亮选区的内容替换

在可视模式下使用 `p` 命令时，vim 将用指定的寄存器内容来替换高亮选区中的文本  

**在替换高亮选区的文本时，vim 做了以下操作**  

1. 从无名寄存器中取出内容
2. 把高亮选区中的内容存入无名寄存器

> 其实就是将 **无名寄存器** 和 **高亮选区** 的内容进行互换  





#### 粘贴出寄存器的内容


普通模式下，根据要插入的文本的性质不同，执行结果也不同  

`p` 命令将寄存器中的文本粘贴到光标之后  
`P` 命令将寄存器中的文本粘贴到光标之前  

对于插入的光标前后的具体位置，需要根据插入的文本的性质决定：  

* 对于插入的文本为一段字符
    * `p`：插入到当前光标的后面
    * `P`：插入到当前光标的前面
* 对于插入的文本为一行字符
    * `p`：插入到当前光标的下一行
    * `P`：插入到当前光标的上一行


面向行的复制或操作，将创建面向行的寄存器  
面向字符的复制或操作，将创建面向字符的寄存器  


> 常用操作：  
> `gp`：在粘贴后光标移动到粘贴内容的最后一行  


---

插入模式下，粘贴寄存器的内容  

> 通常在插入模式中，粘贴面向字符的寄存器  


```text
<C-r>{register}
```

常用操作：  
```text
插入无名寄存器的内容
<C-r>"

插入复制专用寄存器
<C-r>0
```
