## 基本使用


### 文件
> permission 是权限的意思

文件拥有三个基本信息：  

* 拥有者
* 用户组
* 其他人


**用户身份与用户组记录的文件**  
在linux中，默认情况下  

* 系统上的所有账号的相关信息，都记录在 `/etc/passwd` 文件内
* 账号的密码记录在 `/etc/shadow` 文件内
* 用户组组名都记录在 `/etc/group` 文件内


linux系统中 **账号、密码、用户组信息** 都记录在这三个文件中  



#### 权限

| \-rw\-r-\-r-\- | 1      | root       | root           | 1864     | May                  | 4                    | 18:01                | test   |
|:--------------:|--------|------------|----------------|----------|----------------------|----------------------|----------------------|--------|
| 文件类型与权限 | 链接数 | 文件拥有者 | 文件所属用户组 | 文件大小 | 文件最后被修改的时间 | 文件最后被修改的时间 | 文件最后被修改的时间 | 文件名 |

* 文件类型与权限
    * 第一个字符代表这个文件的类型
        * d：目录
        * -：文件
        * l：链接文件
        * b：设备文件里面的可供存储的周边设备（可按块随机读写的设备）
        * c：设备文件里面的串行端口设备（一次性读取设备，如键盘、鼠标）
    * 接下来从左往右：以三个为一组，且均为 `rwx` 的三个参数的组合
        > r：可读（read）  
        > w：可写（write）  
        > x：可执行（execute）  
        > rwx所在的位置不会改变，有权限就会显示字符，没有权限就会显示 -  


        * 第一组为 **文件拥有者的权限** ：可读写，但不可执行
        * 第二组为 **加入此用户组的账号的权限** ：只可读
        * 第三组为 **非本人且没有加入本用户组的其他账号的权限** ：只可读


* 链接数：表示有多少文件名链接到此节点（inode）
* 文件拥有者：表示这个文件或目录的拥有者账号
* 文件所属用户组：表示这个文件或目录所属用户组是哪个
    > 表示在这个文件所属的用户组中的用户，对此文件的权限
* 文件大小：此文件的容量大小，默认单位为Bytes（字节）
* 文件最后被修改的时间：格式（月、日、时间）
    > 如果文件被修改的时间距离现在太久，那么格式中的时间部分仅会显示年份
* 文件名：文件名前面有`.` 的文件，表示隐藏文件



#### 修改权限与属性


**相关命令**  
* chgrp：修改文件所属用户组
* chown：修改文件拥有者
* chmod：修改文件的权限，SUID、SGID、SBIT 等特性



---



**修改所属用户组**  

> chgrp：change group  

**要被修改的组名必须要在 `/etc/group` 中，否则就会报错**  

常用命令及选项  

```bash
chgrp [-R] 用户组名称 文件或目录
```

选项：
* R：进行递归（recursive）修改
    > 连同子目录中的所有文件、目录都更新成为这个用户组  
    > 常用于修改某一目录内所有文件的用户组



---



**修改文件拥有者**  

> chown：change owner  

**对于所修改的文件拥有者，这个用户必须在 `/etc/passwd` 文件中有记录**  

常用命令及选项

```bash
chown [-R] 账号名称 文件或目录
```

```bash
chown [-R] 账号名称：用户组名称 文件或目录
```
> 也可以修改用户的同时修改所属的用户组

选项：
* R：进行递归（recursive）修改
    > 连同子目录中的所有文件、目录都将拥有者更新成为这个用户  
    > 常用于修改某一目录内所有文件的拥有者



---



**修改权限**  

> chmod：change mode  

权限的设置有两种方法：
* 符号修改
    > r：读  
    > w：写  
    > x：执行  

* 数字修改
    > r：4  
    > w：2  
    > x：1  
    > 用数字累加表示对应权限  



---



常用命令及选项（**数字修改**）：
```bash
chmod [-R] xyz 文件或目录
```

选项：
* R：递归修改

参数：
* xyz：数字类型的权限属性（xyz属性数值的相加）


举例：
```bash
chmod 6 test
```

> 6 = 4 + 2 + 0：表示为该文件所设置的权限为读写，但是不能执行



---



常用命令及选项（**符号修改**）：  
```bash
chmod [ugoa][+-=][rwx] 文件或目录
```

* [ugoa]表示操作的对象
    * u：文件拥有者
    * g：文件所属组
    * o：其他人
    * a：所有人（包括：ugo）


* [+\-=]表示操作类型
    * +：添加（为操作对象添加对应权限）
    * -：移除（为操作对象移除对应权限）
    * =：等于（让操作对象的权限等于对应权限）


* [rwx]表示对应权限
    * r：读权限
    * w：写权限
    * x：执行权限


举例：
```bash
chmod u+x,g=r,o-rwx test
```
> 对 `test` 文件执行以下操作  
> * 为文件拥有者添加执行权限
> * 让文件所属组的权限等于只读
> * 为其他人移除对此文件的读、写、执行三个权限

> 用逗号分隔每个操作对象的操作，如果操作对象不需要操作，可以不写  

> 如果操作对象没有此权限，但是命令依旧要删除此权限，shell不会报错  
> 因为最终的结果是一样的  



---



#### 权限的意义

> 权限对应文件和目录的含义是不同的  


--------


**权限对于文件**  

> 文件是实际含有数据的地方  

* r（read）：可读取此文件的实际内容
* w（write）：可以编辑、新增或修改此文件的内容
* x（execute）：此文件有执行权限


**对于文件，w 表示可对此文件的实际内容进行更改**  
**并不包含文件名，因为文件名存储在目录的实际文件中** 
> 这些是文件系统的知识  


--------


**权限对于目录**  

> 目录所存储的是文件名和文件名对应的inode  

* r（read contents in directory）：可读取 **目录的实际内容** （目录的实际内容是该目录下的文件名或子目录，与其对应的inode号）
* w（modify contents of directory）：可更改 **目录的实际内容** （更改此目录的结构，如添加、删除、改名等操作）
* x（access directory）：用户能否进入该目录使之成为 **工作目录**



**如果在目录下不具有 x 的权限，那么就无法切换到该目录下，也就无法执行该目录下的任何命令，即使拥有该目录的 r、w 权限**  

> 如果目录没有 x 权限，那么甚至不能使用命令 cd 进这个目录  

> 如果只有这个目录的 r 权限，但是没有 x 权限，那么就只能读到这个目录的 `文件名结构`  
> 注意只是文件名，关于这个文件的其他信息（比如链接数、文件类型、最后修改时间等）都不能查看，只能查看到这个目录下的文件名结构

> 如果对这个目录有 x、w 权限，但是没有 r 权限，那么就不能查看这个目录下的文件名结构  
> 这里的不能查看目录下的文件结构，是指不能通过 `ls` 等命令来列出当前目录的文件结构（也就意味着不能通过 tab 来补全文件名）  
> 但是可以直接通过指定打开文件名来对文件进行操作  
> 例如：  
> ```bash
> vim te<tab>
> ```  
> 按 tab 键没有用  
> ```bash
> vim test
> ```  
> 但是可以直接通过使用文件全名来指定




#### 文件种类与扩展名

**任何设备在 linux 下都是文件**  


* 常规文件（regular file）
    > 文件类型字符为 `-` 
    * 纯文本文件（ASCII）
    * 二进制文件（binary）
    * 数据文件（data）
        > 有些程序在运行的过程当中会读取某些特定格式的文件，那些特定格式的文件可以被称为数据文件（data file）


---


* 目录（directory）
    > 文件类型字符为 `d`  


---


* 链接文件（link）
    > 文件类型字符为 `l`  


---


* 设备与设备文件（device）
    > 与系统周边及存储等相关的一些文件，通常都集中在 `/dev` 目录下
    * 区块（block）设备文件
        > 文件类型字符为 `b`  
        > 一些存储数据，以提供系统随即存取的接口设备  
        > 例如硬盘等，可以随机地在硬盘的不同区块读写  
        > **也称为快设备**  

    * 字符（character）设备文件
        > 文件类型字符为 `c`  
        > 一些串行端口的接口设备  
        > 例如鼠标、键盘等  
        > 这些设备的特色就是一次性读取，不能够截断输出  


---


* 数据接口文件（sockets）
    > 文件类型字符为 `s`  
    > 通常在 `/run` 或 `/tmp` 目录中  
    > 通常用于`网络上的数据交换`  
    > 网络程序的客户端可以通过这个 socket 文件来进行数据的沟通


---


* 数据传送文件（FIFO, pipe）
    > 文件类型字符为 `p`  
    > 主要用于解决多个程序同时读写一个文件所造成的错误问题  
    > FIFO 是 first-in-first-out 的缩写，也就是管道  



**文件扩展名**  
基本上，linux 的文件没有所谓的扩展名  
> 一个 linux 文件能不能被执行，与它的第一栏的是个属性有关，与文件名没有一点关系  
> **在 linux 中，只要对应用户的权限当中具有 x 的话，就代表这个文件具有可以被执行的能力**  


**但是，使用扩展名有助于我们立即看出这个文件的类型，所以还是会以适当的扩展名来表示该文件是什么种类**  

一般在起文件名的时候，应该可以通过文件名一眼就能看出该文件是干嘛的，所以文件名通常是很长的  





### 目录

#### 目录配置

**linux 目录配置的依据 —— FHS**  
Filesystem Hierarchy Standard（FHS）标准  
> 主要目的是希望用户可以了解到已安装软件通常放置于那个目录下  

FHS 根据过去的经验一直在持续地改版，FHS 依据文件系统使用的频繁与否和是否允许用户随意修改，而将目录定义成为四种交互作用的形态：  

|                | 可分享（shareable）       | 不可分享（unshakeable） |
|----------------|---------------------------|-------------------------|
| 不变（static） | /usr（软件存放处）        | /etc（配置文件）        |
|                | /opt（第三方辅助软件）    | /boot（启动与内核文件） |
| 可变动         | /var/mail（用户邮箱）     | /var/run（程序相关）    |
|                | /var/spool/news（新闻组） | /var/lock（程序相关）   |


* **可分享**：可以分享给其他系统挂在使用的目录
    > 包括可执行文件与用户的邮箱数据等数据，是能够分享给网络上其他主机挂载用的目录  


* **不可分享**：自己机器上面运行的设备文件或是与程序有关的 socket 文件等  
    > 由于仅与自身机器有关，所以就不适合分享给其他主机  


* **不变**：有些数据是不会经常变动的，跟随发行版而不变动
    > 例如函数库、文件说明、系统管理员所管理的主机配置文件等  


* **可变动**：经常修改的数据
    > 例如日志文件、一般用户可自行接收的新闻组等  



**FHS 仅针对目录树架构定义了以下三层目录下应该放置什么数据**：  
* /（root, 根目录）：与启动系统有关
* /usr（unix software resource）：与软件安装/执行有关
* /var（variable）：与系统运行过程有关


FHS 标准建议：**根目录（/）所在分区应该越小越好，且应用程序所安装的软件最好不要与根目录放在同一个分区内，保持根目录越小越好**  
> 如此不但性能较佳，根目录所在的文件系统也较不容易发生问题  
>
> 根目录是整个系统最重要的目录：  
> 1. 所有的目录都是有根目录衍生出来的  
> 2. 根目录与启动、还原、系统修复等操作有关  
>
> 所以 FHS 希望根目录不要放在非常大的分区内，因为越大的分区会放入越多的数据，如此一来根目录所在分区就可能会有较多发生错误的机会  


鉴于上面的原因，FHS 定义出根目录（/）下面应该要有下面这些子目录的存在才好，即使没有物理目录，FHS 也希望至少有链接（link）目录存在才好

| 目录       | 应放置文件内容                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 第一部分： | FHS 要求必须要存在的目录
| /bin       | 系统有很多存放执行文件的目录，但 /bin 比较特殊。因为 /bin 主要放置的是在单人维护模式下还能够被使用的命令。在 /bin 下面的命令可以被 root 与一般账号所使用                                                                                                                                                                                                                                                                                                                                                                                |
| /boot      | 放置启动时会使用到的文件, 包括 linux 内核以及启动选项与启动所需配置文件等                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| /dev       | 在 linux 系统上，任何设备与接口设备都是以文件的形式存在于这个目录当中。只需要通过读写这个目录下面的某个文件，就等于读写某个设备                                                                                                                                                                                                                                                                                                                                                                                                        |
| /etc       | 系统主要的配置文件几乎都放置在这个目录内。<br>一般来说，这个目录下的各文件属性是可以让一般用户查看的，但是只有 root 有权力修改<br>FHS **建议不要放置可执行文件在这个目录中**<br>FHS 还规范了几个重要的目录最好要存在 /etc 目录下：<br>    1. /etc/opt（必要）：放置第三方辅助软件 /opt 的相关配置<br>    2. /etc/X11/（建议）：与 X window 有关的各种配置文件都在这里，尤其是 xorg.conf（X Server 的配置文件）<br>    3. /etc/sgml/（建议）：与 SGML 格式有关的各项配置文件<br>    4. /etc/xml/（建议）：与 xml 格式有关的各项配置文件 |
| /lib       | 放置的是在启动时会用到的函数库，以及在 /bin 或 /sbin 下面的命令会调用的函数库<br>FSH 要求下面的目录必须要存在：    <br>1. /lib/modules/：这个目录主要放置可抽换式的内核相关模块（驱动程序）                                                                                                                                                                                                                                                                                                                                             |
| /media     | 放置可删除设备，包括软盘、光盘、DVD等设备都暂时挂载于此                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| /mnt       | 用来暂时挂载某些额外的设备                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| /opt       | **给第三方辅助软件放置的目录**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| /run       | 早期的 FHS 规定系统启动后所产生的各项信息应该要放置到 /var/run 目录下，新版的 FHS 则规范到 /run 下面，由于 /run 可以使用内存来模拟，因此性能上会好很多                                                                                                                                                                                                                                                                                                                                                                                  |
| /sbin      | **放在 /sbin 下面的为启动过程中所需要的，里面包括了启动、修复、还原系统所需要的命令**linux有非常多命令是用来设置系统环境的，这些命令只有 root 才能够来设置系统，其他用户最多只能用来查询                                                                                                                                                                                                                                                                                                                                                |
| /srv       | srv 可视为 server 的缩写，是一些网络服务启动之后，这些服务所需要的数据目录                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| /tmp       | 是让一般用户或是正在执行的程序暂时放置文件的目录。此目录任何人都可以存取                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| /usr       | 第二层 FHS 设置                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| /var       | 第二层 FHS 设置，主要为放置变动性的数据                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 第二部分： | FHS 建议可以存在的目录                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| /home      | 系统默认的用户家目录                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| /lib<qual> | 用来存放与 /lib 不同格式的二进制函数库，例如支持64位的 /lib64 函数库等                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| /root      | root 的家目录，root 的家目录与根目录应放置在同一个分区，因为如果进入单人维护模式而仅挂载根目录时，该目录就能够拥有 root 的家目录                                                                                                                                                                                                                                                                                                                                                                                                        |


FHS 针对的目录所定义的标准就仅有上面的，不过 linux 下面还有许多非常重要的目录  

| 目录        | 应放置文件内容                                                                                                                                                           |
|-------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| /lost+found | 只有使用标准的 ext2、ext3、ext4 文件系统格式才会产生的一个目录，目的在于当文件系统发生错误时，将一些遗失的片段放置到这个目录下                                           |
| /proc       | 这个目录本身是一个虚拟文件系统，它放置的数据都是在内存中，例如系统内核、进程信息、外界设备的状态及网络状态等。因为这个目录下的数据都是在内存中，所以本身不占任何硬盘空间 |
| /sys        | 与 /proc 类似，也是一个虚拟文件系统，主要也是记录内核与系统硬件相关的内容。包括目前已加载的内核模块与内核检测到的硬件设备信息等                                         |



> 早期 linux 在设计时，若发生问题时，恢复模式通常仅挂载根目录而已，因此有五个重要的目录被要求一定要与根目录放置在一起：  
> * /etc
> * /dev
> * /lib
> * /sbin


**/usr 的意义与内容**  

| 目录           | 应放置文件内容                                                                                                                                                                                                                                                         |
|----------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 第一部分：     | FHS 要求必须要存在的目录                                                                                                                                                                                                                                               |
| /usr/bin       | 所有一般用户能够使用的命令都存放在此，目前已将全部的用户命令放置于此，而使用连接文件的方式将 /bin 连接至此。**另外，FHS 要求此目录下不应该有子目录**                                                                                                                   |
| /usr/local     | 基本上，与 /lib 功能相同，所以 /lib 就是链接到此目录                                                                                                                                                                                                                   |
| /usr/local     | 系统管理员在本机安装自己下载的软件（非发行版默认提供者），建议安装到此目录                                                                                                                                                                                             |
| /usr/sbin      | 非系统正常运行所需要的命令，最常见的就是某些网络服务器软件的服务命令。不过基本功能与 /sbin 差不多，因此目前 /sbin 就是链接到此目录                                                                                                                                     |
| /usr/share     | 主要放置只读的数据文件，也包括共享文件，在这个目录下放置的数据几乎是不分硬件架构均可读取的数据，因为几乎都是文本文件。此目录下常见的还有这些子目录：<br> /usr/share/man：在线帮助文档<br> /usr/share/doc：软件的说明文档<br> /usr/share/zoneinfo：与时区有关的时区文件 |
| 第二部分：     | FHS 建议可以存在的目录                                                                                                                                                                                                                                                 |
| /usr/games     | 与游戏比较相关的数据放置处                                                                                                                                                                                                                                             |
| /usr/include   | c/c++ 等程序语言的头文件（header）与包含文件（include）放置处                                                                                                                                                                                                          |
| /usr/libexec   | 某些不被一般用户常用的执行文件或脚本等，都会放置在此目录中。例如大部分的 X 窗口下面的操作命令，很多都是放置在此目录下                                                                                                                                                  |
| /usr/lib<qual> | 与 /lib<qual> 功能相同，因此目前 /lib<qual> 就是链接到此目录中                                                                                                                                                                                                         |
| /usr/src       | src 有 source 的意思，一般源代码建议放置到这里。至于内核源代码则建议放置到 /usr/src/Linux/ 目录下                                                                                                                                                                      |



**/var 的意义与内容**  
> 如果说 /usr 是安装时会占用较大硬盘容量的目录，那么 /var 就是在系统运行后才会渐渐占用硬盘容量的目录。因为 /var 目录主要针对经常性变动的文件，包括缓存（cache）、日志文件（log file）以及某些软件运行时所产生的文件，包括程序文件（lock file、run file），或例如 MySQL 数据库文件等  


| 目录       | 应放置文件内容                                                                                                                                                                                                |
|------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 第一部分： | FHS 要求必须要存在的目录                                                                                                                                                                                      |
| /var/cache | 应用程序本身运行过程中会产生的一些缓存                                                                                                                                                                        |
| /var/lib   | 程序本身执行的过程中，需要使用到的数据文件放置的目录。此目录下各自的软件应该要有各自的目录                                                                                                                    |
| /var/lock  | 某些设备或是文件资源一次只能被一个应用程序所使用，如果同时有两个程序使用该设备时，就可能产生一些错误的状况，因此就得要将该设备上锁（lock），以确保该设备只会给单一软件所使用，目前此目录已经挪到 /run/lock 中 |
| /var/log   | 日志文件放置的目录，里面比较重要的有 /var/messages、/var/log/wtmp（记录登陆信息）等                                                                                                                           |
| /var/mail  | 放置个人电子邮箱的目录，不过这个目录也被放置到 /var/spool/mail 目录中，通常这两个目录是互为链接文件                                                                                                           |
| /var/run   | 某些程序或是服务启动后，会将它们的 PID 放置在这个目录下，与 /run 相同，这个目录链接到了 /run 目录                                                                                                             |
| /var/spool | 通常放置一些队列数据，所谓的队列就是排队等待其他程序使用的数据，这些数据被使用后通常都会被删除                                                                                                                |


**新老版本的变化**  

* /bin --> /usr/bin
* /sbin --> /usr/sbin
* /lib --> /usr/lib
* /lib64 --> /usr/lib64
* /var/lock --> /run/lock
* /var/run --> /run



**相对路径与绝对路径**  
`.` ：代表当前的目录，也可以使用 `./` 来表示  
`..` ：代表上一层目录，也可以使用 `../` 来代替
